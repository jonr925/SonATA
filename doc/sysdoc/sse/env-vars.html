 General environment variables

There are a number of optional environment variables that can
be used to override various program defaults:

SSE_INITRC - seeker startup file.
    Default: ~/.sserc

SSE_SETUP - directory of config files.
    Default: ~/sse_install/setup

SSE_ARCHIVE - directory root for created data files. 
    Default: ~/sse_archive

SSE_DEBUG_LOG_MAX_FILESIZE_MEGABYTES - max size of sse-debug-log.txt file.
    Default: 25 Megabytes  (Max Value: 500).
    


 Environment variables for configuring component hosts: 


The 'control' commands are executed by the 'controlcomponents' script
in the sse package.  There are a number of environment variables which 
can be used to override the default hosts on which the components
run.  These variables should be set in the environment in which the
runsse script is executed (since runsse in turn executes the
'controlcomponents' script):

1. Ifc host (IF controller):
     Variable: CONTROL_COMPONENTS_IFCx_HOST  

     (where 'x' is the ifc number)

     Example: setenv CONTROL_COMPONENTS_IFC1_HOST "ifcspare"

     Default:  the default host has the same name as the component,
     ie, component "ifc3" will run on the host named "ifc3"

2. Telescope control host:
     Variable: CONTROL_COMPONENTS_TSCOPE_HOST

     All tscope controllers (tscope1, tscope2, etc.) run on the same host.

     Default: the seeker host.


3 ATA Antenna controller host:
     Variable: CONTROL_COMPONENTS_ANT_CONTROL_HOST

     Default: antcntl.atarray.org  (aka 10.3.0.21)

4 PdmArchiver host:
     Variable: CONTROL_COMPONENTS_PDM_ARCHx_HOST

     (where 'x' is the archiver component number).

     Example: setenv CONTROL_COMPONENTS_PDM_ARCH2_HOST "sse4"
     Default: seeker host

5. PPS host:
     Variable: CONTROL_COMPONENTS_PPSx_HOST

     Pulse-per-second time packet broadcast host,
     (where 'x' is the component number)

     Example: setenv CONTROL_COMPONENTS_PPS2_HOST "sse3"

     Default: seeker host
